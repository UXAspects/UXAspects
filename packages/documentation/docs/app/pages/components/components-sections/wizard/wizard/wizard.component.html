The ux-wizard component can be used to present a user with a series of sequential steps. The content of each step should be defined within a ux-wizard-step component.

The following attributes can be used to customize the ux-wizard component:

The current active step. When the step changes an event will be emitted containing the index of the newly active step. If this is not specifed the wizard will start on the first step. Defines whether or not the wizard should be displayed in a horizontal or vertical layout. Defines the text displayed in the 'Next' button. Defines the text displayed in the 'Previous' button. Defines the text displayed in the 'Cancel' button. Defines the text displayed in the 'Finish' button. Defines the text displayed in the tooltip when the 'Next' button is hovered. Defines the text displayed in the tooltip when the 'Previous' button is hovered. Defines the text displayed in the tooltip when the 'Cancel' button is hovered. Defines the text displayed in the tooltip when the 'Finish' button is hovered. If set to true the 'Next' button will appear disabled and will not respond to clicks. If set to true the 'Previous' button will appear disabled and will not respond to clicks. If set to true the 'Cancel' button will appear disabled and will not respond to clicks. If set to true the 'Finish' button will appear disabled and will not respond to clicks. If set to false the 'Next' button will be hidden. If set to false the 'Previous' button will be hidden. If set to false the 'Cancel' button will be hidden. If set to false the 'Finish' button will be hidden. If set to true the 'Cancel' button will be visible even on the last step. By default it will be hidden on the final step. If set to true the 'Finish' button will be visible on all steps of the wizard. By default this button will only be visible on the final step of the wizard. Emits before the current step changes. The event contains the current step index in the from property, and the requested step index in the to property. Emits when the current step has changed. Emits when the wizard has moved to the next step. It will receive the current step index as a parameter. Emits when the wizard has moved to the previous step. It will receive the current step index as a parameter. Emits when the 'Cancel' button has been pressed. Emits when the 'Finish' button is clicked, but before the finish event fires. This fires regardless of the validity of the final step. Emits when the 'Finish' button has been pressed and the final step is valid.

The following properties can be used to configure the ux-wizard-step:

The text to be displayed in the wizard step tab. Allows you to define whether or not a step is valid. The user will not be able to proceed to the next step if this property has a value of false. Defines whether or not this step has previously been visited. A visited step can be clicked on and jumped to at any time. By default, steps will become 'visited' when the user navigates to a step for the first time. Emits when visited changes.
