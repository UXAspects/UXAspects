The Popover directive can be used to show extra content when required. This content can be a string of text or your own custom template and it supports multiple placements.

In our CodePen example we have created a popover which appears when hovering the email addresses. This popover shows more detail about the sender and recipients and contains a nested popover which the user can view by clicking the button.

There are 2 versions of the popover which you can implement:

  • popover - takes a string of text only and will escape any HTML provided.
  • popover-template - takes a string variable which specifies the location of a template to be used for the popover body.

The Popover directive also provides some optional attributes to configure how it will be displayed:

Name Type Binding Description Optional
popover-title string literal A string to display as the popover title. true
popover-placement string literal Describes where the popover should be placed. Available options are top, bottom, left and right. Defaults to top. true
popover-animation boolean literal Describes whether the popover should fade in/out. Defaults to true. true
popover-popup-delay number literal Describes how long the delay (in milliseconds) should be before the popover appears. Defaults to 0. true
popover-trigger string literal Describes the event which will trigger the popover to appear. The options are:
  • mouseenter - popover will display while the element is hovered.
  • click - popover will display when the element is clicked and will hide when the element is clicked again.
  • focus - popover will display when the element is focused and will hide when the element is blurred.
  • outsideClick - same as click only clicking anywhere outside of the directive element or popover will hide the popover.
  • insideHover - same asmouseenter only the popover will not disappear while the directive element or popover are hovered.
 true
popover-inside-delay number literal Describes how long the delay should be before the popover disappears while the popover or directive element are not hovered. Only required when popover-trigger="insideHover" is used. Defaults to 500 milliseconds. true
popover-append-to-body boolean literal Describes whether the popover should be appended to the body instead of the parent element. Defaults to true. true
popover-class string literal Custom class to be applied to the popover. true

Note: If you wish to implement a nested popover inside a popover which has set popover-trigger="insideHover" then you must ensure the nested popover has set popover-append-to-body="false" otherwise both popovers will disappear when hovering the nested popover.
