The ux-date-time-picker component can be used to select a date/time or timezone. It can be displayed as a standalone component or within an ngx-bootstrap popover.

The following options can be used to configure the appearance and behavior of the component.

Name Type Description
@Input @Output date Date The selected date to be displayed in the component. This will also emit an event when the date is changed using the component.
@Input @Output timezone DateTimePickerTimezone Will set the selected timezone. If not defined the users current timezone will try to be used. If their timezone is not available it will revert to GMT. This will also emit an event if the selected timezone is changed.
@Input timezones DateTimePickerTimezone[] Defines the list of available timezones. The DateTimePickerTimezone interface specifies that each timezone should be an object with a name property that represents the timezone, eg. GMT+2, and an offset property that represents the number of minutes relative to GMT the timezone is.
@Input showDate boolean Defines whether or not the date picker should be visible.
@Input showTime boolean Defines whether or not the time picker should be visible.
@Input showTimezone boolean Defines whether or not the time picker should allow the user to choose a timezone.
@Input showSeconds boolean Defines whether or not the time picker should allow the user to specify seconds.
@Input showMeridian boolean Defines whether or not the time picker should show an AM/PM button, or time should be represented in 24hr format instead.
@Input showSpinners boolean Defines whether or not the time picker should allow the user to select the time using spinners.
@Input weekdays string[] If defined will override the weekday names displayed.
@Input nowBtnText string Defines the text to be displayed in the button used to set the selected time to the current time.
