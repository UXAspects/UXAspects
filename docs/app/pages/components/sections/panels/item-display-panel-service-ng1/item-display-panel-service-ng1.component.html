The Item Display Panel Service may be used to show and hide the Item Display Panel without using the directive. This is useful when more programmatic control over the display panel is required.

An example of this service is shown in the Tree Grid documentation.

To use the service, inject $displayPanel into the controller. This provides the following functions:

Name Description
open Opens the display panel with the specified options, or updates the contents if it is already open. It takes the following arguments:
  • element - the HTML element which is being displayed, e.g. tr element in a table.
  • itemDisplayPanelOptions - the options for the display panel, as documented in the Item Display Panel section.
  • shadow - a boolean argument determining whether a shadow effect appears underneath the title header of the display panel.
  • previous - a boolean argument indicating whether there is a previous item to switch to; useful if navigation buttons are used within the display panel.
  • next - a boolean argument indicating whether there is a next item to switch to; useful if navigation buttons are used within the display panel.
close Closes the display panel if it is open. It takes the following argument:
  • itemDisplayPanelOptions - the options for the display panel, as documented in the Item Display Panel section.
panelOpen Returns true if the panel is open.
panelHidden Returns true if the panel is hidden. If the animate setting is used then the panel will be hidden instead of closed.
getCurrentPanel Returns the element with which the open function was called.
movePrev

Puts the browser focus on the previous sibling of the element which was passed to the open function, if there is one. Note that this function will not directly update the display panel contents. It takes one optional argument:

  • selector - a jQuery selector expression to match elements against.
moveNext Puts the browser focus on the next sibling of the element which was passed to the open function, if there is one. Note that this function will not directly update the display panel contents. It takes one optional argument:
  • selector - a jQuery selector expression to match elements against.
