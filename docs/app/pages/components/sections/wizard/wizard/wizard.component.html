The ux-wizard component can be used to present a user with a series of sequential steps. The content of each step should be defined within a ux-wizard-step component.

The following attributes can be used to customize the ux-wizard component:

Name Type Description
@Input @Output step number The current active step. When the step changes an event will be emitted containing the index of the newly active step. If this is not specifed the wizard will start on the first step.
@Input orientation 'vertical' | 'horizontal' Defines whether or not the wizard should be displayed in a horizontal or vertical layout. The default value is horizontal.
@Input nextText string Defines the text displayed in the 'Next' button.
@Input previousText string Defines the text displayed in the 'Previous' button.
@Input cancelText string Defines the text displayed in the 'Cancel' button.
@Input finishText string Defines the text displayed in the 'Finish' button.
@Input nextTooltip string Defines the text displayed in the tooltip when the 'Next' button is hovered.
@Input previousTooltip string Defines the text displayed in the tooltip when the 'Previous' button is hovered.
@Input cancelTooltip string Defines the text displayed in the tooltip when the 'Cancel' button is hovered.
@Input finishTooltip string Defines the text displayed in the tooltip when the 'Finish' button is hovered.
@Input nextDisabled boolean If set to true the 'Next' button will appear disabled and will not respond to clicks.
@Input previousDisabled boolean If set to true the 'Previous' button will appear disabled and will not respond to clicks.
@Input cancelDisabled boolean If set to true the 'Cancel' button will appear disabled and will not respond to clicks.
@Input finishDisabled boolean If set to true the 'Finish' button will appear disabled and will not respond to clicks.
@Input nextVisible boolean If set to false the 'Next' button will be hidden.
@Input previousVisible boolean If set to false the 'Previous' button will be hidden.
@Input cancelVisible boolean If set to false the 'Cancel' button will be hidden.
@Input finishVisible boolean If set to false the 'Finish' button will be hidden.
@Input cancelAlwaysVisible boolean If set to true the 'Cancel' button will be visible even on the last step. By default it will be hidden on the final step.
@Input finishAlwaysVisible boolean If set to true the 'Finish' button will be visible on all steps of the wizard. By default this button will only be visible on the final step of the wizard.
@Output onNext number This event will be fired when the 'Next' button is pressed. It will receive the current step index as a parameter.
@Output onPrevious number This event will be fired when the 'Previous' button is pressed. It will receive the current step index as a parameter.
@Output onCancel void This event will be fired when the 'Cancel' button is pressed.
@Output onFinish void This event will be fired when the 'Finish' button is pressed.

The following properties can be used to configure the ux-wizard-step:

Name Type Description
@Input header string The text to be displayed in the wizard step tab.
@Input valid boolean Allows you to define whether or not a step is valid. The user will not be able to proceed to the next step if this property has a value of false.
@Input @Output visited boolean Defines whether or not this step has previously been visited. A visited step can be clicked on and jumped to at any time. By default, steps will become 'visited' when the user navigates to a step for the first time.
