Flippable Cards are a neat and tidy way to show more information about something, without taking up screen space.

To define the content that should be displayed on the front and back sides of the card you should add html to ux-flippable-card-front and ux-flippable-card-back components that are placed inside the ux-flippable-card tags.

You can customize the appearance and behavior of Flippable Cards by using the following attributes:

Name Type Description
@Input @Output flipped boolean Determines whether or not the card is flipped. If two way binding is used this value will be updated when the state of the card changes.
@Input direction string Determines whether the card should flip horizontally or vertically. Default value is horizontal.
@Input trigger string Determines when the card should flip. Possible options are click, hover and manual. The manual option should be used if you want complete control over when the card should flip. Default value is hover.
@Input width number Sets the width (in pixels) of the card. The default value for this is 280.
@Input height number Sets the height (in pixels) of the card. The default value for this is 200.
