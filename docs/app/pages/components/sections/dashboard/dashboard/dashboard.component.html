The ux-dashboard component allow you to easy create a customizable dashboard for your application. Simply add ux-dashboard-widget elements inside the ux-dashboard tag and the dashboard component will automatically arrange the widgets for you.

The dashboard layout can be manually configured using the layout attribute. This is a two way binding which will be updated whenever the layout changes.

In the example above all widgets can be dragged to reposition them. This is achieved by adding a ux-dashboard-widget-drag-handle attribute to an element within the widget. Whichever element has the attribute is the one the user has to perform the drag on.

Widgets can also be resized, by setting the resizable attribute to true on a ux-dashboard-widget component. Widgets are given two classes widget-col-span-* and widget-row-span-* with the * being replaced by the number of columns and rows the widget is occupying. This can be used to apply different styles based on the size of the widget.

If the dashboard becomes too small to display widgets without them becoming smaller than the specified minimum size, the dashboard will display in stacked mode where all items are presented in a vertical list and will span the entire width of the dashboard.

The following attributes can be added to the ux-dashboard component to configure its behavior:

Name Type Description
@Input @Output layout DashboardLayoutData[] If defined or changed this will set the positions of the widgets within the dashboard. This is a two way binding that will be updated with the current layout when it changes.
@Input options DashboardOptions Configures the options for the dashboard, if an option is not specified the default value will be used.

The following DashboardOptions are available to configure:

Name Type Description
columns number Defines the number of columns the dashboard should have.
minWidth number Defines minimum widget width.
minHeight number Defines minimum widget height.
rowHeight number Defines the height of each row on the dashboard.
emptyRow boolean Defines whether or not an empty row should always be visible on the bottom of the dashboard to allow for additional items or dragging/resizing.

The following attributes can be added to the ux-dashboard-widget components to configure their behavior:

Name Type Description
@Input id string Sets the ID of the widget. Each widget should be given a unique ID.
@Input colSpan number Defines the number of columns this widget should occupy.
@Input rowSpan number Defines the number of rows this widget should occupy.
@Input resizable boolean Defines whether or not this widget can be resized.
