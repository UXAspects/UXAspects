The Preview Pane Window allows you to open a separate window for the currently displayed file. This is especially useful when using dual monitors allowing a larger preview window to be viewed on a second monitor.

You can have a list of files and when traversing through the list with the Preview Pane Window open, you can see the page updating to display the correct file.

A very interesting feature of the Preview Pane Window is that when you are displaying the preview in a separate window, any changes made in the secondary window will update the original window, and vice versa. This means you can see your changes happening live and don’t have to worry about writing complex code to keep the 2 windows in sync (2 way communication between the windows achieved).

You can see this in our example above by navigating to any .html file (e.g. Document 1.html) and clicking the “Standalone Viewer” button above the Preview Pane area ( ). Enter text into the “Signed” input field and you will see the Preview Pane Window updating as you type, and vice versa if you make changes in the Preview Pane Window.

In order to implement the Preview Pane Window, you must use the preview-pane-window directive and give it a unique preview-name attribute. This will insert the “Standalone Viewer” button which the user can click to open the Preview Pane Window. The preview-pane directive must also have a preview-name attribute set which will be the same value as the attribute you set in the preview-pane-window directive.

Refer to the Preview Pane page for further documentation on how to implement a Preview Pane.
