The safeInterval service is a scope-safe way to execute repeating functions in your code. It provides a wrapper to the Angular $interval service that will automatically cancel your intervals when their scope is destroyed, preventing the memory leaks, race conditions and unexpected behaviour that can result from intervals which have not been properly destroyed.

To use the safeInterval service, inject it into your controller or directive in the same way you would normally inject the Angular $interval service. Then call the constructor to get an instance that will be local to your execution context. The constructor takes the $scope variable as a parameter - this is used to listen for the $destroy event raised by Angular.

The returned instance has two methods.

Name Description
interval Two parameters, a function to execute and an optional delay in milliseconds to specify how often the function should repeat. Returns an interval object.
cancel One parameter, an interval object to be cancelled.
