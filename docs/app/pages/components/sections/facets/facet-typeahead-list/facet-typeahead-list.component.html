The Facet Typeahead List component can be used to select and deselect facets to provide the user with filtering capabilities. The Facet Typeahead List component is particularly useful when there can be a large number of facets that can be selected and allows users to find the appropriate facet by searching using a typeahead control.

The typeahead can support an array of facets to filter or if there is a significant amount of data, an observable can be used to fetch data from the server when the user performs a search.

To add a Facet Typeahead List component to your application you need to import the FacetsModule which allows the use of the ux-facet-typeahead-list component.

The following attributes can be used to customise the behavior of the component:

Name Type Description
@Input facets Facet[] | Observable<Facet[]> Defines the complete list of facets that can be selected. Alternatively an observable can be used to allow values to be fetched dynamically. The search query is available on the destination.outerValue property on the observer.
@Input @Output selected Facet[] This will allow you to define an initial set of selected facets. If two-way binding is used this array will get updated any time the selected facets change.
@Output events FacetEvent This will be triggered when a facet is selected, deselected or all facets are deselected. The event will be an instance of either FacetSelect, FacetDeselect or FacetDeselectAll and will contain the facet being selected or deselected in a facet property (deselect all will not contain affected facets).
@Input header string Defines the text displayed at the top of the Facet Typeahead List.
@Input typeaheadConfig FacetTypeaheadListConfig Allows configuration of the typeahead control. The possible values are:
  • placeholder - string - Sets the placeholder of the typeahead.
  • minCharacters - number - Defines the minimum number of characters are required before results will be shown. Default: 1.
  • maxResults - number - Sets the maximum number of results to display. Default: 5.
  • delay - number - Defines the number of milliseconds to wait the results are filtered. Default: 0.
@Input expanded boolean Defines whether or not the Facet Typeahead List should be initially expanded or not. Default: true.
