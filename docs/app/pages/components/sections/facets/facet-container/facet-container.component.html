Facets can be used to easily implement a way for users to filter a set of data, allowing them to apply and remove conditions. Facet selection components are usually contained within a Facet Container component. A Facet Container provides an overall view of all facets that are applied, and allow the user to quickly remove an individual facet, or all facets, without having to go through each facet selection component.

To add a Facet Container component to your application you need to import the FacetsModule which allows the use of the ux-facet-container component.

The following attributes can be used to customise the behavior of the component:

Name Type Description
@Input @Output facets Facet[] Allows a predefined set of Facets to be displayed. If using two-way binding this array will update when the selected facets change.
@Output events FacetEvent This will be triggered when a facet is selected, deselected or all facets are deselected. The event will be an instance of either FacetSelect, FacetDeselect or FacetDeselectAll and will contain the facet being selected or deselected in a facet property (deselect all will not contain affected facets).
@Input header string Defines the text displayed at the top of the Facet Container. Default: Selected:.
@Input clearTooltip string Defines the text to display in the tooltip when hovering over the clear all button. Default: Clear All.
@Input emptyText string Defines the text to display when there are no selected facets. Default: No Items.

Facet Object

A facet object can contain the following properties:

Name Type Description
title string This is the text that will be displayed when the facet is shown.
data any? Allows custom data to be stored on the facet which can be used when performing any filtering.
count number? Defines the number of items in the data set that match this facet. Facet selection components can display this if provided.
disabled boolean? Defines if the facet can be selected. Some selection components may grey disabled options out, others may not display the item.
id string? This can be useful if multiple facets may share the same title. An ID can be used to ensure each facet is unique.

These properties can be defined by passing the values to the constuctor in the order listed above.
