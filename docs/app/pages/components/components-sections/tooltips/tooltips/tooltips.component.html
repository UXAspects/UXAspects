To add a tooltip to an element add the uxTooltip attribute with the required text. The tooltip is placed at the top by default.

The following attributes can be used to configure the uxTooltip directive:

If a string is specified, the text will be displayed as the tooltip content. If a TemplateRef is provided the content of the template will be inserted into the tooltip. If set to true the tooltip will not be shown by any of it's triggers. If set, the provided class will be added to the tooltip. This can be used to change the appearance of the tooltip. If set, this will alter the role of the tooltip. If a TemplateRef is used as the content this can be used to provide the template with data. If specified, the tooltip will wait the specified number of milliseconds before showing. If set to true this will programmatically show the tooltip. Defines which direction the tooltip should appear. The possible values are top, right, bottom and left. Specifies which events will cause the tooltip to appear. To manually control when the tooltip appears set this to an empty array. Possible events are click, mouseenter and focus. Specifies which events will cause the tooltip to hide. To manually control when the tooltip hides set this to an empty array. Possible events are click, mouseleave and blur. This event will be triggered when the tooltip is shown. This event will be triggered when the tooltip is hidden. This event will be triggered when the tooltip is shown or hidden and will be provided will a boolean representing the visibility of the tooltip. Open the tooltip. Hide the tooltip. Toggle the visibility of the tooltip.
