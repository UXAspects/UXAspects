The media player component can be used to view video and audio files. It supports a range of the most popular video and audio file formats. A comprehensive list of supported formats in each browser can be found here.

To add a media player to your page use the ux-media-player selector. The following attributes can be used to configure the appearance and behavior of the media player:

Defines the appearance of the media player. The two possible values are video and audio. The media player will adapt it's appearance to best suit the type specified. The url to the media file to be loaded by the media player. If enabled, the controls in the media player will be hidden unless the mouse is over the player and will appear in a darker style. Dark mode is automatically enabled in full screen mode. Quiet mode is only available for videos. The anonymous keyword means that there will be no exchange of user credentials when the media source is fetched. If specified the function will be called passing the current volume as an argument. It should return an appropriate aria label for the mute/unmute button. If specified the function will be called passing the current playing state as an argument. It should return an appropriate aria label for the play/pause button. If specified the function will be called passing the current fullscreen state as an argument. It should return an appropriate aria label for the fullscreen toggle button. If specified the function will be called passing the current track as an argument. It should return an appropriate aria label for the subtitle selection button. Define an aria label for the go to start button. Define an aria label for the go to end button. Define an aria label for the title displayed in the subtitle selection popover. Define an aria label to indicate subtitle are not currently enabled. Define an aria label to indicate there are no subtitles available. Define an aria label for the media player. Define an aria label for the the seek element.

To provide subtitles or closed captions add an HTML track element as a child of the ux-media-player. More information on the track element can be found here.

Custom controls can be added to the media player by adding a child element with a uxMediaPlayerCustomControl attribute. To programmatically control the media player you can either:

  • Create a component and inject the MediaPlayerService.
  • Use a template reference variable to access the MediaPlayerService on the ux-media-player component.
