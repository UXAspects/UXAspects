The Contacts control can be used to display a shorthand notation for the participants in a communication. The participant’s name, short name or initials can be shown, along with an organization, and individuals belonging to the same organization can be grouped together. The status of a contact, active or passive, can be displayed to indicate their involvement in a conversation. Organizations can be labelled as “External” or “Risk”.

To use the Contacts control use the contact-group directive. The full list of configuration attributes for this control is defined below:

An array of contacts to display (format defined below) or a single contact object. An object defining the organization (format defined below). The size of the control, either medium or small. An object mapping the primary, secondary, active, and passive colors in #HTML format. The primary color will be used as the background for the contact. We recommend you use colors from the Color Palette. The maximum number of contacts to display. The remainder will be condensed into an overflow segment. Handler for the click event in the overflow segment.

Contact Object

Contacts can be supplied either as a single object, or more typically as an array of objects.

The name to be displayed for the contact. Either "active" or "passive". Simple tooltip to show text or HTML for the contact on hover. This can be used to design a complex tooltip which appears on hover. The customTooltip object can have the following attributes :
  • template: (required) The relative path to the HTML template to be displayed for the contact on hover.
  • tooltipPosition: (optional) either "top" or "bottom" - default value is "bottom".
  • data: (optional) The data attribute can be defined as an object to contain information for the tooltip. The values in the data object can be displayed accordingly through the template HTML.

The example at the top of the page shows usage of the ‘active’ status, the ‘passive’ status, and no status.

Organization Object

Organizations should be objects with the following properties:

The name to be displayed for the organization. String, either external or risk. Simple tooltip to show text or HTML for the organization on hover. This can be used to design a complex tooltip which appears on hover. The customTooltip object can have the following attributes :
  • template: (required) The relative path to the HTML template to be displayed for the contact on hover.
  • tooltipPosition: (optional) either "top" or "bottom" - default value is "bottom".
  • data: (optional) The data attribute can be defined as an object to contain information for the tooltip. The values in the data object can be displayed accordingly through the template HTML.

The example at the top of the page shows usage of both the “risk” and “external” labels.
