Hotkeys can be added to elements by adding window:keydown to the host property of your component. This will trigger the specified function any time a key is pressed on the window offering a wide range of uses.

The above example shows two different example behaviours - on the left, the hotkey tabs between each item in the group, including the group itself. In this case the user can ‘press’ the button using the spacebar or associated default keybinding for the control - this will be unchanged.

In the example on the right, the hotkey tabs between each item in the group and also immediately invokes the behavior associated with that button.

Both groups display the ‘hotkey hint’ in their upper-left corner. This can be used to inform the user of available keys and is optional.

When using the hotkey-hint, the precise styling can be controlled using the hotkey-group-hint class. This class exists in UX Aspects and will show a hotkey hint in the style shown in this page. To customize the appearance of the hotkey hint, you can override this class with your own values. As an example of this - on this page, the default styling has been extended to provide a top and left margin of -2px on the hotkey hint, because the example button panels above use a 2px width border and we want them to appear flush. This also allows you to change the appearance of the hotkey hint depending on the component. For example, the demo above is wrapped with a hotkey-demo-hint class, so defining a CSS class .hotkey-demo-hint .hotkey-group-hint allows us to override the CSS for just this instance.

Usage Notes

This example uses some styles which are purely for illustrative purposes; however the ‘focus’ style is important to provide. UX Aspects supplies the aspects-focus-outline class which you may place on your container element; this will apply the same ‘focus’ style as other controls in UX Aspects. If required you may write your own class to change the styling of your element using the :focus CSS pseudo-selector.
