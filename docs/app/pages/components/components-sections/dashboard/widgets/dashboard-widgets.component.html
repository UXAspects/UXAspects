The pre-built widgets which include the ux-dashboard-actions-widget, ux-dashboard-enum-widget, ux-dashboard-table-widget and ux-dashboard-text-widget are ready-to-use widgets for the ux-dashboard component.

The following attributes are required by all pre-built widgets:

The id given to the widget. The name used by aria labels and announcements. The heading displayed for the widget. Whether the widget can be dragged and resized. The number of columns the widget spans. The number of rows the widget spans.

Each widget has its own customizable attributes.

Actions Widget

The ux-dashboard-actions-widget displays a status of a system and provides the user options to react to the status.

It has the following additional attributes:

The buttons that will be displayed. Includes label (a string), icon (a ux-icon code), iconTemplate (a TemplateRef) and action (a function). The status displayed by the widget.

Additionally the actions widget has these outputs available:

Emits when the selected value has been changed and saved.

Enum Widget

The ux-dashboard-enum-widget is a select input at its core. The label of the currently selected value is displayed in the dropdown input. The value can be changed by using the dropdown input.

The enum widget has the following additional attributes:

Additionally the enum widget has these outputs available:

Emits when the selected value has been changed and saved.

Table Widget

The ux-dashboard-table-widget is a normal HTML table element which renders a header and a data array into a scrollable table.

The table widget has the following additional attributes:

The headers for the table columns. The data being displayed in the table. Each sub-array should be the same length as the header array.

Text Widget

The ux-dashboard-text-widget displays static text. The button in the bottom right opens a side panel which also fills the whole parent dashboard component and displays the full text. Additionally the side panel can be set to be a text editor as well so that the user can edit the text and save it.

It has the following additional attributes:

The text that is being displayed. Whether the text can be edited in the side panel. An object that contains the localized labels for the component.

Additionally the enum widget has these outputs available:

Emits when the text has been changed.
