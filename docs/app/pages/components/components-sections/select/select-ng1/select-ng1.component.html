The dynamic-select component allows users to choose one or more items from a set via a dropdown interface. It supports filtering and keyboard control. For very large data sets, a paging feature is available, ensuring that data is only loaded as required.

The data set may be a collection of strings or objects. In the above example, the single select controls on the left use an array of strings as a data source, while the multiple select controls on the right are set up to select objects. The upper controls use a fixed array to provide the items to select from, whereas the lower controls use a paging function which contains a delay to simulate a server request. This can be observed in action by scrolling or using the down key to reach the end of the current result set in the select dropdown.

The following attributes are available to configure the dynamic-select directive.

Specifies that the control should allow multiple selection; otherwise only a single item may be selected at a time. The selected item(s). Provides the items that the user may select from. Items may be strings or objects. Use select-as and track-by to present objects as required. If an array is provided then the full contents will be displayed in the dropdown. There must be no identical items. If a function is provided, this will retrieve pages of items as required when scrolling or filtering the item dropdown. The arguments are as follows:
  • pageNumber (numeric) - The index of the requested page, starting from 0.
  • pageSize (numeric) - The number of items requested.
  • searchString (string) - The text entered by the user to filter the results.
The function should return either an array, or if the data is retrieved asynchronously, a promise containing an array. Return an empty array if there are no more items. If the source provides objects, specify the object property name which will contain the text to be displayed in the control. If the source provides objects and the select-as property is not unique, specify the object property to use as a unique key. Additional configuration options for the control, see below for available properties. Determines whether the control is disabled.

The options attribute receives an object with the following properties:

The informational text to display in the input area when no text has been entered. If true the placeholder will only be shown when no items are selected. This only applies when multiple selection is enabled. Determines whether the selected value may be set to null by deleting or editing the text in single selection mode. Determines whether the dropdown has a fixed height and scrolls. This is always true when a paging function is supplied, but may be set to false for small, fixed data sets to ensure that the dropdown fits the displayed items. The number of items per page to request. Applicable only when using a paging function in the source attribute. The direction in which the dropdown list expands from the control. "up" specifies that list appears above the input area, and "down" specifies that it appears below.
