The select-table component can be used for single select in a table format. The component can be defined using a select-table element. The options for the select component should be passed as attributes as explained below:

Array of strings or Array of objects, this attribute should be passed with the values to be shown in the select table. When the options are passed as an array of objects, this attribute should be set to the name of the key, whose value from the object will be displayed. Note that this should only be set when the options are passed as an array of objects, and not required when the options are passed as an array of strings. The selected option will be set to the variable passed to the selected attribute. The component can be associated with a search text which can filter the options. The search-text attribute can be set to the text with which options will be filtered. The height of the table can be changed by providing a string to the table-height attribute (e.g. "50%"). This can be set to px, % etc. Controls the behaviour when selected items are filtered from view. By default filtered items remain selected. Set to the string 'clear' to clear the selected item when the search-text filter has removed it from the list of visible items. Alternatively, set to 'reselect' to both clear the selected item, and reselect it when it becomes visible again. Customize the content of each item by providing a template. The scope of the template will be the item data. Customize the content of each item by providing a template url. The scope of the template will be the item data.
