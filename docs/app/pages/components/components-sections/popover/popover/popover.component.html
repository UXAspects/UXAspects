To add a popover to an element add the uxPopover attribute with the required text. The popover is placed at the top by default.

The following attributes can be used to configure the uxPopover directive:

If a string is specified, the text will be displayed as the popover content. If a TemplateRef is provided the content of the template will be inserted into the popover. If set to true the popover will not be shown by any of it's triggers. If set, the popover will display a title above as the popover content. If set, the provided class will be added to the popover. This can be used to change the appearance of the popover. If set, this will alter the role of the popover. If a TemplateRef is used as the content this can be used to provide the template with data. If specified, the popover will wait the specified number of milliseconds before showing. If set to true this will programmatically show the popover. Defines which direction the popover should appear. The possible values are top, right, bottom and left. Specifies which events will cause the popover to appear. To manually control when the popover appears set this to an empty array. Possible events are click, mouseenter and focus. Specifies which events will cause the popover to hide. To manually control when the popover hides set this to an empty array. Possible events are click, mouseleave, blur, escape and clickoutside. This event will be triggered when the popover is shown. This event will be triggered when the popover is hidden. This event will be triggered when the popover is shown or hidden and will be provided will a boolean representing the visibility of the popover. Open the popover. Hide the popover. Toggle the visibility of the popover.
