Column sorting can be added to a table by applying the uxColumnSorting directive to the tr element and adding the ux-column-sorting component in the th of any columns that are sortable.

If set to true the column will sort by only this column, removing sorting from all others. A template can be provided to customize the appearance of the sort indicator.

The component will be exported as ux-column-sorting.

Defines a unique identifier for the column that can be used when sorting. Defines the sorting direction of a column: NoSort, Ascending or Descending. When manually setting this value it is the responsibility of the application to ensure that the table sort state matches the column sort state. Defines the sorting order of a column. When manually setting this value it is the responsibility of the application to ensure that the table sort state matches the column sort order. Defines if a column can be toggled to the NoSort state. If set to false this will toggle between the Ascending and Descending states when changing the sort direction. Emits when state changes. Emits when the sort order changes. Changes the state of the sorting on the column between NoSort, Ascending and Descending. This returns an array of objects for each column being sorted containing key: string and state: ColumnSortingState. State can be used to find the current sorting state of the column eg. (state === ColumnSortingState.Ascending). The ColumnSortingOrder interface has been provided for objects in the array. Defines the sorting order of a column: NoSort, Ascending or Descending. If multiple columns are being sorted this will contain the sort order of the column.

The .column-sorting-header class can be used to align the sorting icon to the right of the table header as seen in the example above.
