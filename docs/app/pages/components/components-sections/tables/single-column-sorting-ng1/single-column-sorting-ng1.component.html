The different options to sort are contained in the sortable-header directive. The sortable-header will require one of the following classes to be added in order to set the text size for the headers.

  • sorter-header-sm - sets the font size to 14px.
  • sorter-header-md - sets the font size to 16px.
  • sorter-header-lg - sets the font size to 18px.

If using this in conjunction with the scrollable table one of the following classes is required.

  • scrollable-header-sm - sets the font size to 14px.
  • scrollable-header-md - sets the font size to 16px.
  • scrollable-header-lg - sets the font size to 18px.

The following attributes will need to be supplied to the sortable header:

Array of objects (defined below) used to store all the required header information. String indicating which column will be sorted by default when the page loads, setting this to a variable will allow the sort to be remembered between visits. Boolean option indicating the sort order of the default column when the page loads, setting this to a variable will allow the sort to be remembered between visits. Boolean option indicating if the sort icon should appear immediately after the text instead of being positioned at the end of the column.

The controller implements the select function and will accept the following attributes within the headers object:

The column header. True if the column can be used as a sorter. Sort expression for the column. True if the column header should be centered. A class for the header in order to set the column to a fixed width. This results in columns that do not change size when a sort has been applied or according to data in the cells. The following classes have been created to allow for fixed column widths:
  • table-col-icon - width of 65px. Optimum width for a column containing icons.
  • table-col-xs - width of 5%.
  • table-col-sm - width of 10%.
  • table-col-md - width of 20%.
  • table-col-lg - width of 30%.
  • table-col-xl - width of 40%.
  • fixedLayout - sets the table to a fixed layout. To be used in conjunction with the classes above and should be placed within the table tag.
Note: when applying a fixedClass to a column header you must also apply that same class to the td (data row) for that column. You should also specify the class in the table tag to be fixedLayout.
