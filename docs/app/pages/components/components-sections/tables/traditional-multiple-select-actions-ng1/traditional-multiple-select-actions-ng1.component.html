These are actions that can be performed against multiple items in a table or a ul with the .card-view class applied. Traditional selection has a hit area only inside the checkbox for each item. Selection mode will be activated when at least one check box is selected.

Multiple selection actions should be inside a div.affix-toolbar which should be inside a div.affix-container. Use the multiple-list-select-actions element to wrap all the actions. The multiple-list-select-actions has the following attributes:

Should take in an item as a parameter. This is called with the selected item when it is selected. on-select="vm.select(item)" Should take in an item as a parameter. This is called with the selected item when it is deselected. When clearing all selections this method will be passed null as the item. on-deselect="vm.deselect(item)" Should take in an item as a parameter. This is called when it is necessary for the directive to uniquely identify an item. key-fn="vm.getId(item)" Should return the total count of items. This is used for the Select All function. total="vm.totalItems"

All HTML elements that need to be displayed in the toolbar when list is not in selection mode are enclosed in div .disp. The div .disp should also contain any other HTML elements that are not actions but need to be displayed in the toolbar. All HTML elements that need to be displayed in the toolbar when the list is in selection mode are enclosed in div .reverse-disp. The elements inside div .disp can include multiple-select-action, which can also trigger the selection mode.The multiple-select-action can have following attributes:

The name of the action. This is called when the user confirms they want to proceed with the action. Used to style the action button. Values that can be given here are primary, accent, secondary and warning.

The div .reverse-disp can have elements display-select-options and display-cancel-option. The display-select-options will display the Select All/None options and count of selected items. The display-cancel-option will show the cancel button which will discard the selection mode on click.

To get the check box, you must use the .multi-select-checkbox class. Also, multiple-list-select-item="item" attribute should be set to the specified item you want to select. Note the value specified for the attribute is evaluated against the current $scope. The tr or li element should have the classes .clickable .single-select. When using a table the extended-checkbox-hit attribute can be added to the td element that encloses multiple-list-select-item to extend the hit area to the space around the checkbox.

A selection can be made invalid by changing the items in the grid. A selection can only be made invalid if “Select All” was clicked to select all items. The items are considered to have changed when the count, specified in the total attribute on the multiple-list-select-actions, has changed.

multipleSelectProvider can be used to set the text that is displayed by these directives. You can set the following properties on the multipleSelectProvider:

Displayed beside the "x" when in selection mode. Displayed when no items are selected, should encourage user to select items. Displayed when items are selected. Displayed beside the item count when more than one item is selected. Displayed beside the item count when a single item is selected. Displayed in the link for selecting all. Displayed in the link for selecting none. Displayed when the selection becomes invalid.
