These are actions that can be performed against multiple items in a table or a ul with the .card-view class applied.

Multiple selection actions should be inside a div.affix-toolbar which should be inside a div.affix-container. Use the multiple-select-actions element to wrap all the actions. The multiple-select-actions should also contain any other HTML elements that are not actions but need to be displayed in the toolbar. The multiple-select-actions has the following attributes:

Should take in an item as a parameter. This is called with the selected item when it is selected. on-select="vm.select(item)" Should take in an item as a parameter. This is called with the selected item when it is deselected. When clearing all selections this method will be passed null as the item. on-deselect="vm.deselect(item)" Should take in an item as a parameter. This is called when it is necessary for the directive to uniquely identify an item. key-fn="vm.getId(item)" Should return the total count of items. This is used for the Select All function. total="vm.totalItems"

To allow items to be selectable when in selection mode you must have the multiple-select-item="item" attribute set to the specified item you want to select. If using a table you should put this attribute on the tr, or if being used with .card-view then put this attribute on the li. Note the value specified for the attribute is evaluated against the current $scope. A checkbox can be added to the item with the multiple-select-checkbox class and the attribute ng-model="multipleSelectChecked". The td should also be given the multiple-select-checkbox-table class.

A selection can be made invalid by changing the items in the grid. A selection can only be made invalid if "Select All" was clicked to select all items. The items are considered to have changed when the count, specifed in the total attribute on the multiple-select-actions, has changed.

multipleSelectProvider can be used to set the text that is displayed by these directives. You can set the following properties on the multipleSelectProvider:

Displayed beside the "x" when in selection mode. Displayed when no items are selected, should encourage user to select items. Displayed when items are selected. Displayed beside the item count when more than one item is selected. Displayed beside the item count when a single item is selected. Displayed in the link for selecting all. Displayed in the link for selecting none. Displayed when the selection becomes invalid.
