Wizards are defined inside a wizard element. Each step is defined using a step element. Use the step-heading element to define the html for the step heading. Note that the number of the step is automatically prepended to the HTML. Everything not inside the step-heading is considered the content of the step. An overflow-tooltipcan be added to the wizard and will require max-heightand max-width to be added to the element. Alternatively a single-line-overflow-tooltipcan be added to the wizard which will require a max-width to be set and vertical-alignto be set to bottom.

Attributes that can be defined on the wizard element are:

Defines the text, tooltip and visibility of a button.
  • nextText - (default:"Next") - defines the text for the 'Next' button.
  • previousText - (default:"Previous") - defines the text for the 'Previous' button.
  • cancelText - (default:"Cancel") - defines the text for the 'Cancel' button.
  • finishText - (default:"Finish") - defines the text for the 'Finish' button.
  • showNext - (default: true) - defines whether or not to show the 'Next' button.
  • showPrevious - (default: true) - defines whether or not to show the 'Previous' button.
  • showCancel - (default: true) - defines whether or not to show the 'Cancel' button.
  • showFinish - (default: true) - defines whether or not to show the 'Finish' button.
  • nextTooltip - optional attribute that allows you to set the tooltip for the 'Next' button.
  • previousTooltip - optional attribute that allows you to set the tooltip for the 'Previous' button.
  • cancelTooltip - optional attribute that allows you to set the tooltip for the 'Cancel' button.
  • finishTooltip - optional attribute that allows you to set the tooltip for the 'Finish' button.
  • hideCancelOnFinish - (default: true) - defines whether or not the show the 'Cancel' button when on the final (finish) step.
  • previousEnabled - (default: true) - defines whether or not the 'Previous' button is enabled.
  • nextEnabled - (default: true) - defines whether or not the 'Next' button is enabled.
  • cancelEnabled - (default: true) - defines whether or not the 'Cancel' button is enabled.
  • finishEnabled - (default: true) - defines whether or not the 'Finish' button is enabled.
  • alwaysShowFinish - (default: false) - defines whether or not the 'Finish' button is shown in all the steps.
Expression that is called when changing steps. The expression should take in two parameters, from and to, which are the indexes of the steps involved in the change. If it returns false then the current step goes into error state and the step change is stopped. e.g. on-changing="canChange(from,to)". The expression can also return a number representing a step index to move to, overriding the to parameter. Expression that is called when Finish is clicked but before on-finished expression is evaluated. The expression should return a boolean result. If false current step goes into error state and the step change is stopped. e.g. on-finishing="canChange()". Expression that is called when Finish is clicked, after on-finishing expression is evaluated. e.g. on-finished="wizardComplete()". Expression that is called when Cancel is clicked. e.g. on-canceled="wizardCanceled()". Set to indicate if a wizard is already visited(edited) and to enable all the steps.
