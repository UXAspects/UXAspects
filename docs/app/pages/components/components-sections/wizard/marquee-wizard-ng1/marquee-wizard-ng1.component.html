Attributes that can be defined for the Marquee Wizard:

Specifies the icon for the wizard. Specifies the behaviour of each step:
  • title - String for the title of the step.
  • html - HTML code for the content of the step.
  • header - String for the title of the wizard step.
  • templateUrl - Template for the content of the wizard step.
  • stepTemplateUrl - Template for the content of the side panel wizard step. The template scope will contain the properties of the associated step.
  • hidden - When set to true the step will be hidden from the marquee wizard
  • error - When set to true the step will be highlighted in red.
  • completed - This value will update as each step is completed but can be overwritten by setting the boolean value.
  • visited - This value will update as each step is visited but can be overwritten by setting the boolean value.
Defines the text, tooltip and visibility of a button.
  • nextText - (default:"Next") - defines the text for the 'Next' button.
  • previousText - (default:"Previous") - defines the text for the 'Previous' button.
  • finishText - (default:"Finish") - defines the text for the 'Finish' button.
  • showNext - (default: true) - defines whether or not to show the 'Next' button.
  • showPrevious - (default: true) - defines whether or not to show the 'Previous' button.
  • showFinish - (default: true) - defines whether or not to show the 'Finish' button.
  • nextTooltip - optional attribute that allows you to set the tooltip for the 'Next' button.
  • previousTooltip - optional attribute that allows you to set the tooltip for the 'Previous' button.
  • finishTooltip - optional attribute that allows you to set the tooltip for the 'Finish' button.
  • previousEnabled - (default: true) - defines whether or not the 'Previous' button is enabled.
  • nextEnabled - (default: true) - defines whether or not the 'Next' button is enabled.
  • finishEnabled - (default: true) - defines whether or not the 'Finish' button is enabled.
  • visibilityOnStep - (default: undefined) - define a function which can override the visibility of the previous, next, and finish buttons on a per-step basis. This receives the step index as an argument, and should a return an object. On the object, configure the button(s) to be shown or hidden by specifying the boolean properties previous, next, and finish. Leave the property unset to continue using the default show/hide behavior for the respective button.
Expression that is called when changing steps. The expression should take in two parameters, from and to, which are the indexes of the steps involved in the change. If it returns false then the current step goes into error state and the step change is stopped. e.g. on-changing="canChange(from,to)". The expression can also return a number representing a step index to move to, overriding the to parameter. Expression that is called when Finish is clicked but before on-finished expression is evaluated. The expression should return a boolean, number or a promise that resolves to a boolean or number. If the function or promise returns a number the wizard will go to that step. If the function or promise returns false current step goes into error state and the step change is stopped. e.g. on-finishing="canChange()". Expression that is called when Finish is clicked, after on-finishing expression is evaluated. e.g. on-finished="wizardComplete()". Expression that is called when Cancel is clicked. e.g. on-canceled="wizardCanceled()". Set to indicate if a wizard is already visited(edited) and to enable all the steps. Object to define the side panel info between the icon and the steps.
  • title - optional string to define the title in the side info panel.
  • description - optional string to define the description in the side info panel.
