The splitter directive is used to add a container for the resizable panels. The splitter can present the panels horizontally or vertically. Inside the container the panels should be specified using the splitter-panel directive.

The following attributes can be added to the splitter directive to configure its behavior.

Should be set to either 'horizontal' or 'vertical' depending on the desired orientation. Specifies the thickness, in pixels, of the divider between panels. The default value is 10. The minimum recommended width is 4. If the user resizes a panel within this amount of pixels of its minimum size it will automatically snap to the minimum size. Specifies a function to be called when the user begins dragging the divider. Specifies a function to be called when the divider is being moved. Specifies a function to be called when the user finishes dragging the divider. Specifies a function to be called when the user expands or collapses a side inset splitter panel. It will be passed a value of true if it is being expanded and false if it is collapsing.

The following attributes can be added to the splitter-panel directive to configure its behavior.

Specifies the initial size of the panel as a percentage. The size of all panels combined should total 100. A total higher than this will cause overflow. Specifies the desired minimum size of a panel in pixels.

Full Height Usage

In order to have the splitter take up the remaining height of the page, you should set its height to 100% and ensure the full page hierarchy of containers down to that point also have their heights set to 100%. If you wish to remove the scrollbar from the window itself and have only the splitter panel internals be scrollable, then you should set the height of the element containing the splitters to be 100% minus your header height. Currently this is only supported using the condensed header.
