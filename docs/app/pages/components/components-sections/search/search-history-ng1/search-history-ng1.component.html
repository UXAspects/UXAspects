Search History can be saved and updated with the use of the localStorage object. Data stored here will have no expiration time and can be easily accessed from any page when required.

Please note that when using localStorage you should include a fallback option to use cookies or if you are running a hybrid application you can use the PersistentDataService.

Using our above example you can click the icon to see details of the last 10 searches.

The localStorage object has the following methods which we use in our example above:

Name Description
setItem This method takes 2 parameters, a key name and key value pair, and will add the key to the storage or update it if it already exists.
getItem This method takes a key name as a parameter and returns its value (if it exists).

Note: Refer to the Storage API for full documentation
