The ux-date-time-picker component can be used to select a date/time or timezone. It can be displayed as a standalone component or within an ngx-bootstrap popover.

The following options can be used to configure the appearance and behavior of the component.

The selected date to be displayed in the component. Will set the selected timezone. Defines the list of available timezones. The DateTimePickerTimezone interface specifies that each timezone should be an object with a name property that represents the timezone, eg. GMT+2, and an offset property that represents the number of minutes relative to GMT the timezone is. Defines whether or not the date picker should be visible. Defines whether or not the time picker should be visible. Defines whether or not the time picker should allow the user to choose a timezone. Defines whether or not the time picker should allow the user to specify seconds. Defines whether or not the time picker should show an AM/PM button, or time should be represented in 24hr format instead. Defines whether or not the time picker should allow the user to select the time using spinners. If defined will override the weekday names displayed. Defines the text to be displayed in the button used to set the selected time to the current time. Emits an event when the date is changed using the component. If not defined the picker will try to use the user's timezone. If that is not available, it will revert to GMT.
