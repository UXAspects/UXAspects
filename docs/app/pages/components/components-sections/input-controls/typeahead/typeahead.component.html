The ux-typeahead component can be used to apply a dropdown typeahead list to a text input.

To match the width of the input control, place both inside a container with position: relative.

The following properties can be used to customize the appearance and behavior of the ux-typeahead component:

If an array is provided, this is the list of options which can be chosen from. It can be an array of strings or custom objects, matching the format used by the tag input control. If custom objects are required, the display property must also be set. If a function is provided, this is used as a callback to dynamically retrieve options in pages. The parameters are:
  • pageNum: number - The index of the requested page, starting from 0.
  • pageSize: number - The number of items requested.
  • filter: T - The filter details as provided via the filter binding.
The return value should be either a promise which resolves to an array, or a plain array in case the data can be loaded synchronously. An empty array or an array with fewer than pageSize items can be returned, which indicates that the end of the data set has been reached. The string which is used to filter the list of options. This is matched to the display property if custom objects are used. The tag input input binding can be used here, to refine the typeahead as the user enters text. Controls whether the typeahead dropdown is visible. Determines the display value of the options, if they are custom objects. This may be a function or a string. If a function is provided, it receives the option object as an argument, and should return the appropriate display value. If the name of a property is provided as a string, that property is used as the display value. Determines the unique key value of the options, if they are custom objects. This is only relevant if multiple options will have the same display value. This may be a function or a string. If a function is provided, it receives the option object as an argument, and should return the appropriate key value. If the name of a property is provided as a string, that property is used as the key value. The default behaviour is to use the display value as the key. Set the currently active item. The list of options which should appear disabled in the typeahead. This is useful if duplicate tags are not wanted in the tag input, in which case the tags binding should also be set here. The positioning of the typeahead dropdown in relation to its parent. The maximum height of the typeahead dropdown, as a CSS value. Whether the parent control allows multiple selection. This is used to inform assistive technologies. A template which will be rendered in the dropdown while options are being loaded. A loading state to be specified. Whether the dropdown should appear when the filter appears. A template which will be rendered in the dropdown for each option. The following context properties are available in the template:
  • option: T - the string or custom object representing the option.
  • api: TypeaheadOptionApi - provides the functions getKey, getDisplay and getDisplayHtml. See the section below for more information.
A template which will be rendered in the dropdown if no options match the current filter value. The number of options to request in a page. This should ideally be more than twice the number of items which fit into the height of the dropdown, but this is not required. Controls whether the first non-disabled option in the dropdown is highlighted by default. Controls whether an option should be selected on enter key press. List of recently selected options. Number of displayed recently selected options. Emits when open is changed. Raised when an option on the dropdown is clicked. The option property on the event contains the clicked option. Raised when the highlighted option changes, providing the currently highlighted option. Raised when the highlighted option changes, providing the currently highlighted DOM element. Emits when the list of recently selected options changes.

TypeaheadOptionApi

The api object available to the optionTemplate has the following interface:

Returns the unique key value of the given option. Returns the display value of the given option. Returns the display value of the given option with HTML markup added to highlight the part which matches the current filter value. Override the ux-filter-match class in CSS to modify the default appearance.
