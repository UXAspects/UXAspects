Each item that opens the item display panel is contained in the display-panel-item directive.

The shadow property is optional and can be added to the display-panel-item to display a curved shadow under the header.

The auto-index attribute is used to enable keyboard controls - full documentation can be found on the Table Indices page.

The input value of the directive is an object. It is defined by the panel property of each item object in vm.items which in turn defines the item display panel.

The following defining the modal are set in the panel property of each item.

If you have a modal that appears over the top of a display panel ensure you add ng-mouseup="$event.stopPropagation()" to the modal element and any buttons within the modal to prevent the display panel from closing when mouse events occur. Alternatively, setting the outsideClick property to false in the display panel options can also be used.

Defines the Header title of the item display panel. Provides a path to a template representing item display panel content. Provides a path to a template representing item display panel footer content. Classes defining the number of columns as per the grid system for a responsive item display panel or a class with a preset width. Defines the initial top position of the modal with respect to standard header, condensed header or toolbar. Element id or class, item display panel is displayed in reference to this element after scroll. A scope instance to be used for the modal's main content. A boolean value to set the panel to animate in and out of the screen. A boolean value to indicate whether or not clicking outside of the panel should close it.

The next and previous buttons can be added to the panel footer to navigate between the items. The ng-click attribute of the previous button should be set as the previous() function and the ng-click attribute of the next button should be set as the next() function. The previous and next buttons will be disabled accordingly if the ng-disabled attribute is set to previousBtnStatus and nextBtnStatus respectively.
