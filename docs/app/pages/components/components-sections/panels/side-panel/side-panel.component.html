The ux-side-panel component can be used to create a panel which can be displayed or hidden as needed. It can be attached to the browser window, or any other container such as a modal.

Add the uxSidePanelClose directive to a button to attach a close button in the top right corner of the side panel.

Content can be added freely within ux-side-panel; however, if a header and/or footer are required, the following classes can be used to structure the content of the side panel:

  • ux-side-panel-header
  • ux-side-panel-content
  • ux-side-panel-footer

The following properties can be used to configure the ux-side-panel component:

Whether the side panel is open (visible). Whether the panel displays inline with other content in the same container. This ensures that no content is hidden by the panel. If this is set, the panel's position will be defined by its parent container, and opening the panel will change the layout of the other content to make space for the panel. Which parent to visually attach the panel to. Has no effect if inline is true.
  • window: the panel will appear relative to the browser window.
  • container: the panel will appear relative to the nearest positioned parent (i.e. position is relative, absolute, or fixed). Use this option to place a side panel inside a modal.
The number of pixels to offset the top of the panel. This can be used to display the panel below a page header. Has no effect if inline is true. Whether to display the side panel as a modal. As a modal, user input is restricted to the contents of the side panel. Has no effect if inline is true. Whether to animate the panel opening and closing. Whether the panel should close when the user clicks outside the panel. Emitted when the side panel opens or closes. Opens the side panel. Closes the side panel.
