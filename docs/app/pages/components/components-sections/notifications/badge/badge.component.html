The ux-badge directive can be used to easily attached a badge to any DOM element to be used as a label/tag, or to show (x) number of new items relating to it etc.

Note: the badge directive relies on the CSS overflow property being set to visible for the subject element due to how the badge element attaches to it. If you want to use hidden, you will need to add a wrapper element within the subject so it only afters the contents.

The appearance and behavior of the directive can be configured using the following attributes:

The color you want to use for the background color. This should be a string value that matches one of the colors available in the ColorService. Sets a character / number limit on the content within the badge. If the content is a number and larger than this value it will display this value with an appended+, otherwise the content will be treated as text and use this value to truncate the number of characters displayed followed by an ellipsis (…). The text you wish to used for accessibility devices, such as screen readers etc. If not set this will default to the badgeText value. The vertical position of the badge in relation to it's parent element. The horizontal position of the badge in relation to it's subject element. This will determine whether the badge content will overlap it's subject element. If overlap is false the badge will start after the end of the subject element, otherwise if it is set to true the badge horizitonal center will be at the end of it's subject element. The size of the badge. This visibility of the badge.
