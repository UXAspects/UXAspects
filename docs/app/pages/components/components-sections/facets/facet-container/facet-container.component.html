Allow reordering

Facets can be used to easily implement a way for users to filter a set of data, allowing them to apply and remove conditions. Facet selection components are usually contained within a Facet Container component. A Facet Container provides an overall view of all facets that are applied, and allows the user to quickly remove an individual facet, or all facets, without having to go through each facet selection component.

To add a Facet Container component to your application you need to import the FacetsModule which allows the use of the ux-facet-container component.

The following attributes can be used to customise the behavior of the component:

Allows a predefined set of Facets to be displayed. Defines the text displayed at the top of the Facet Container. Defines the text to display in the tooltip when hovering over the clear all button. Defines the text to display when there are no selected facets. Determines if the facets can be reordered. If using two-way binding this array will update when the selected facets change. This will be triggered when a facet is selected, deselected or all facets are deselected. The event will be an instance of either FacetSelect, FacetDeselect or FacetDeselectAll and will contain the facet being selected or deselected in a facet property (deselect all will not contain affected facets).

Facet Object

A facet object can contain the following properties:

This is the text that will be displayed when the facet is shown. Allows custom data to be stored on the facet which can be used when performing any filtering. Defines the number of items in the data set that match this facet. Facet selection components can display this if provided. Defines if the facet can be selected. Some selection components may grey disabled options out, others may not display the item. This can be useful if multiple facets may share the same title. An ID can be used to ensure each facet is unique.
