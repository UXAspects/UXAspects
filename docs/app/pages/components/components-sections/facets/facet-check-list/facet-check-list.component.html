The Facet Check List component can be used to display a list of Facets that can be selected or deselected to provide the user with filtering capabilities. This component is designed to be placed within the Facet Container component.

To add a Facet Check List component to your application you need to import the FacetsModule which allows the use of the ux-facet-check-list component.

The following attributes can be used to customise the behavior of the component:

Defines the complete list of facets that can be selected. This will allow you to define an initial set of selected facets. Defines the text displayed at the top of the Facet Check List. If false the list will grow to display all possible facets. If true a scrollbar will appear to prevent the list from growing too large. Defines whether or not the Facet Check List should be initially expanded or not. Defines whether or not the checkboxes will appear in simplified form. Sets the HTML id of the component. For accessibility purposes, the id of the child component ux-facet-check-list-item will be this value with the -check-list-item suffix. If two-way binding is used this array will get updated any time the selected facets change. This will be triggered when a facet is selected, deselected or all facets are deselected. The event will be an instance of either FacetSelect, FacetDeselect or FacetDeselectAll and will contain the facet being selected or deselected in a facet property (deselect all will not contain affected facets).
