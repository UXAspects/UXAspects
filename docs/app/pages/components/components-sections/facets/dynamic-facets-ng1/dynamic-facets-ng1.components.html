When there are a large number of facet options, the Dynamic Facet can be used to replace the regular scrollable list with a text search, supporting dropdown typeahead. This is configurable to fallback to the standard scrollable list, when the number of items in the list falls beneath a certain threshold.

To use the Dynamic Facet, use the facet-dynamic directive instead of the facet directive. Set the facet-options attribute with an object containing a title attribute, and an options property containing collection of objects that follow the interface for the facet-option defined above.

Note: For this type of facet, however, each one of these objects is accessible via the data attribute. Since both name and count are properties within that object, they should be accessed as data.name and data.count, respectively.

The Dynamic Facet can be used in the same facet-container as the standard facet directive.

To configure the behaviour of the control, set the display-configuration attribute, which expects an object of the following format:

How many characters should be typed before a dropdown is shown. How many entries to show in the typeahead dropdown. How many Facet Options should be displayed regardless of whether a typeahead is visible. Items shown as Facet Options will not be listed in the typeahead dropdown. When there are fewer than this many items, a list of Facet Options will be shown instead of a text input. When true any Facet Options with a count of zero will be visible.

The maximum number of characters expected by this text area is defined by the max-lengthdirective.

The maximum number of lines the text input is allowed to grow to can be set using the max-lines attribute. If the number of lines exceeds this value then a scrollbar will appear.

To prevent the user from taking a new line using the return key set the disable-return attribute to false.

The values presented in the dropdown can also be changed dynamically. This is optional behaviour, but useful if you wish to perform additional server requests on demand. The function supplied to update-callback will be triggered every time the contents of the text input are modified. The function will be called with:

  • textInput - the text value in the search control.
  • optionList - the array of options the control can filter from.

The function must return:

  • optionList - the array of options, with the contents modified, added or removed as required.

Set the placeholder attribute to a string property to set the placeholder text that will be shown in the text search input control.

If desired, the user has the ability to specify a different template for all dynamic facet options by setting the facet-option-template attribute to a template.
