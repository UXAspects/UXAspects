The safeTimeout service is a scope-safe way to execute delayed functions in your code. It provides a wrapper to the Angular $timeout service that will automatically cancel your timers when their scope is destroyed, preventing the memory leaks, race conditions and unexpected behaviour that can result from timers which have not been properly destroyed.

To use the safeTimeout service, inject it into your controller or directive in the same way you would normally inject the Angular $timeout service. Then call the constructor to get an instance that will be local to your execution context. The constructor takes the $scope variable as a parameter - this is used to listen for the $destroy event raised by Angular.

The returned instance has two methods.

Name Description
timeout Two parameters, a function to execute and an optional delay in milliseconds. Returns a timer object.
cancel One parameter, a timer object to be cancelled.
