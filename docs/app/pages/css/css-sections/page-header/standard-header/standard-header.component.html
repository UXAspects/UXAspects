The page header contains the product name and logo, application switcher, a text search box and links on the right hand side. There should be no more than two links on the right hand side.

The content of the page header should be placed in the following html.

Add affix-element directive to fix the header when a page scrolls. The affix-element directive must appear inside a div.affix-container.

The product name, logo, application switcher and the text search box are contained in the .navbar-header. This is how it should be structured:

“Search…” text box is contained in the expand-input directive, explained in Input Expand.

The links are contained in .navbar-top-links with the addition of .navbar-right to right align its content.

The feature to switch applications is contained in the standard header within the product-styling class. It is contained in the application-switcher-container directive.

Each application is contained in the application-switcher-item directive which provides the following options:

This will be the application name. This function expression will be called when an application option is selected. This function should perform application switching. This defines an abbreviated value used to recognize the application option.

A link can be a simple anchor tag:

Alternatively it can be a dropdown containing a list of options. This can be useful for displaying notifications as demonstrated on this page.

When using the links it is also important to add the correct classes to the .nav-header so that the search text input can take up the appropriate width. .top-link should be added when only one anchor link with text is present. You can use .top-link-2 if you are using 2 anchor links with text. .top-link-dropdown should be added when using a link with a dropdown and .top-link-dropdown-2 when using 2.

Add expand-search directive to the search icon (which appears when the header is condensed) to expand and collapse the search text entry.
