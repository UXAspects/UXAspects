The breadcrumb trail may also include tab state on a single page. This can be achieved by creating a UI Router state for each tab, and connecting the states with the active tab in a controller. The breadcrumb may then display a different breadcrumb text for each tab, and child views may be associated with particular tabs for the purposes of navigation.

To configure this, create the tab view using the select and active attributes.


In the above example, the patterns.pagelayout.breadCrumbWithTabState. documentationDetail state acts as a child of patterns.pagelayout.breadCrumbWithTabState. documentationTab in the breadcrumb trail, even though it is at the same level of hierarchy. This is due to the ncyBreadcrumb.parent property, and allows it to load in the same ui-view as the parent.

Note: If you use the ncyBreadcrumb.parent property, the Back Button will require additional code to navigate correctly. See the Back Button section of the Standard Header documentation for an appropriate implementation of the target-state attribute.
