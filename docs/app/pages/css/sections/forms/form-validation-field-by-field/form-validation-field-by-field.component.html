The form above demonstrates validation of a field’s content as the user enters the field.

The user will be prompted if the field contains an error and may only click the ‘Submit’ button when the entire form contains no errors (see the ng-disabled attribute on the button in the HTML below).

The form layout is created with the use of classes such as form-group, form-label etc. which are described in our ‘Basic Form’ documentation.

Notice how any div tag which contains an input that requires validation implements the form-group-validate and has-feedback (used to show warning icon) classes. Inputs that require validation must have their name and ng-model attributes set.

A required attribute can be specified if the input is required.

The input type attribute also specifies our desired type such as text, email etc. and the input will only be valid when the users input matches this.

To display a ‘warning’ icon we add the form-control-feedback class in a span tag. We then use the ng-show attribute to only show the warning icon when required.

For example, to access the validity of a desired input, specify the form name followed by the input name. You can then access the inputs properties (states) such as $valid, $touched etc. using the properties provided by ng-model. This can be seen in the HTML code sample below.
