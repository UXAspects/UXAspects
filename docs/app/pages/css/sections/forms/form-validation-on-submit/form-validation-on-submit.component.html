The form demonstrated above can be used to validate each field when the user clicks the ‘Submit’ button. The user will be shown any affected fields with focus being on the first affected field.

If you wish you use a Form with Validation on submit, any div tag which contains an input that requires validation must implement the form-group-validate-on-submit class, rather than the ‘form-group-validate’ class used for field-by-field validation.

Inputs requiring validation must set the ng-class attribute, applying the on-submit class only when the form has been submitted i.e. submitted=true. submitted is only set to true when the ‘Submit’ button has been clicked, as seen in our HTML sample below.

When the submit button is initially clicked and there are fields with errors, it will be disabled until each field is corrected.

Note: you must set the ng-class attribute and only apply the disabled class when the form has been submitted but still has errors ($invalid), for this to work correctly.

See the above documentation on field-by-field validation for information on how to show error text and error icons, however you must also add submitted to the ng-show expression to only show these when the form has been submitted.
