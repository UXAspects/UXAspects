The Sankey Chart allows transfers and flows within a system to be presented visually.

To create a Sankey chart, use the sankey directive.

The data for the chart should be set with the chart-data attribute. There are two optional attributes, a click attribute for handling interaction, and an options attribute for configuration of the chart.

Chart-Data

Sankey Charts are composed of a number of nodes representing areas of a system. These are arranged into columns and connected by links. The required format for the chart data is as shown:

Each column should be defined with a name and an id. The value given for ordinal defines the order in which the columns will be presented.

Node items should have a name and id. The type property should correspond to the id of a column. The value property should hold the value for the node. The example code also uses another property, datasize, which represents file size in bytes. Additional properties such as this can have arbitrary names; they will be mapped to internal properties depending on the options object which is supplied. Additional properties supplied must be numeric.

The links should have a source and target, which must correspond to the ids of the nodes which need to be connected. Each link in the chart requires a definition of this kind. Links can also have data associated with them, which will affect their size on the chart. If the value of a node is higher than the total of the values of the links exiting that node, the excess values will be represented with an overflow link (shown above as the orange links exiting the Not Classifed and Classified nodes in the example).

When defining links, ensure that only nodes in adjacent columns are connected.

To convey chart information to assistive technologies, include a text summary of the data. Use the .sr-only class to make the text hidden but visible to screen readers, and set the tabIndex attribute on the element so it can be reached with the keyboard.

Options

The configuration object for the Sankey Chart allows data properties to be mapped to positions on the chart nodes.

Name Type Description Optional
linkHoverHL boolean Determines whether to have a link hover highlight. Default is true. true
linkTooltip object Configuration for tooltips on the Sankey ribbons. See the linkTooltip object definition below. true
col object Configuration for the column header. See the col object definition below. false
block object Configuration for the chart blocks. See the block object definition below. true
overflow object Configuration for the overflow links. See the overflow object definition below. true
showColNumber boolean Determines whether the column number is shown in the header. Default is false. true

The linkTooltip object is defined as follows:

Name Type Description Optional
show boolean Determines whether the tooltip is shown. Default is true. true
label string Text to show in the tooltip, as a suffix to the numeric value. true

The col object is defined as follows:

Name Type Description Optional
paddingTop number Space in pixels between the header and the top of the chart column. true
paddingBottom number Space in pixels at the bottom of the chart. true
headerLabelLength number The number of characters after which column content is truncated. false
headerLabelSpacing number The number of pixels between the column number (if visible) and the label. false

The block object is defined as follows:

Name Type Description Optional
minWidth number Minimum width of the block in pixels. This depends on the container and columns, if the value is set too low it will revert to a default value. true
minHeight number Minimum height of the block in pixels. Setting this to smaller values will show a greater differentiation in box size. true
calloutData object Configuration for the values shown in the blocks. See the calloutData object definition below. true

The calloutData object is defined as follows:

Name Type Description Optional
topLeft object Configuration for the top left section of the blocks. See the callout section object definition below. true
topRight object Configuration for the top right section of the blocks. See the callout section object definition below. true
bottomRight object Configuration for the bottom right section of the blocks. See the callout section object definition below. true
bottomLeft object Configuration for the bottom left section of the blocks. See the callout section object definition below. true

The callout section (topLeft, topRight, bottomRight, bottomLeft) object is defined as follows:

Name Type Description Optional
key string Specifies the key of the value from the data source to be displayed. false
nodeLabel boolean Set to true for the value which defines the name of the node. This will be truncated to ensure it fits inside the block. true
defaultShow boolean Determines whether the data will be visible without hover. Default is true. true
label string Describes the unit of measurement for the data. true
valueUnit string Provides a data-formatter for certain data types - the letter supplied here will receive a prefix (K, M, G, etc.) as appropriate and the associated value will be scaled. In the above example all values for file size are given in bytes and automatically formatted to GB, MB etc. by supplying the value B. true
binary boolean Set to true to use binary prefixes (e.g. k = 1024) instead of decimal (k = 1000). This is particularly useful for showing file sizes. true
valueFormatter function Allows the value to be formatted as desired. The argument to the function is the numeric value, and it should return a formatted string. This may be useful for locale presentation, using the toLocaleString function. true
click function Allows an on click function to be added to the callout and returns the data. true

The overflow object is defined as follows:

Name Type Description Optional
tooltip string Configuration for the tooltip on overflow links. See the overflow.tooltip object definition below. true

The overflow.tooltip object is defined as follows:

Name Type Description Optional
showTooltip boolean Determines whether a tooltip will be shown on overflow links. Default is true. true
label string Text to show in the tooltip, as a suffix to the numeric value. true

The Sankey chart will grow and shrink in size where possible to fit the space available to it. To have further control over the size of the chart provide styling for the .sankey class.

Responsiveness

The Sankey Chart has certain limitations when it comes to responsiveness. This is because the blocks must have a minimum width and height in order to accurately portray data. If the data set consists of multiple columns/blocks then the chart will only be responsive above a specific width/height. This varies depending on the number of columns/blocks and the block.minWidth and block.minHeight properties specified in the chart options.
