The partition map can be used to provide a visual representation of a hierarchy structure.

The partition map appearance and behavior can be customized using the following:

The root segment of a tree structure defining the partition map. See below for details of the PartitionMapSegment interface. Define the colors to be used for each row and the order they should appear. Define the currently selected item. Determine the percentage height of collapsed segments. Define the minimum desired pixel width for a segment. Segments will appear at least this width regardless of their value providing there is sufficient available space. Define the function that will return the aria announcement for a segment. This will be announced when a segment receives focus. By default the focused node name will be announced along with the names of its ancestors. Emits whenever a segment is selected.

PartitionMapSegment Interface

Define the name of the segment. Define the child segments if there are any. If there are no children a value property must be provided. Define the value this segment represents. This should only be present on segments with no children. Optionally provide some additional information about this segment. This can be accessed in a custom segment template and used to display additional information.

Additionally the content of a segment can be customized by adding a child ng-template identified as #partitionMapSegment. This custom template can be used to add functionality such as a tooltip or popover. The template context provides access to several variables that can be used to display information within the segment:

This variable contains the segment data. It will also contain any additional custom data that was provided in the data property. The amount this segment represents. If this segment has children, the value will represent the sum of all child values. The background color of this segment.
