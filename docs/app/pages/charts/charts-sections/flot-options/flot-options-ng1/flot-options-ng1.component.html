Listed below are the options which can be specified in the controller for a chart:

Defines the chart type and its layout. Values include 'lines', 'bars', 'points', 'pie' etc. E.g. If data represents a bar chart, then the value is set to 'bars' etc. Configures the X-axis with options like show, color, etc. Configures the Y-axis with options. Defined with "top", "left", "bottom" and "right", space in pixels between the canvas edge and the grid. Markings, used to draw simple lines and rectangular areas in the background of the plot. It can be an array of ranges in the form of { xaxis: { from, to }, yaxis: { from, to } } or a function(above example) that returns this range (note: markings are only used for Live Charts). Specifies colors used to represent a chart. Helps configure the grid options. Defines the data series labels and label boxes with the color of the series. Defines how a tooltip should be shown. Called when plot area is clicked. Parameters are event, the position and a nearby data item object. This option works when the "clickable" attribute is set to true. Called when plot area is hovered. Parameters are event, the position and a nearby data item object. This option works when the "hoverable" attribute is set to true. If "autoHighlight" is true (the default), nearby data items are highlighted automatically.

The following attributes should be defined within series

If set to true the chart is displayed. Specifies the colors of the fill. If more than one color is specified this will produce a gradient. If set to true fills the chart with colors in the fillColors property. A number between 0 and 1 defines the opacity of the fill. Defines the thickness of the line or outline in pixels. Centers the axis labels if set to `center`. Default color of the translucent overlay used to highlight on mouse hover. Defines the fill color for the portion representing forecast data. Defines the stroke color for the portion representing forecast data. Default color of the translucent overlay used to highlight on mouse hover for the portion representing forecast data. Defines a list of numbers that specifies distances to alternately draw a line and a gap (in coordinate space units). Similar to this property in HTML5 Canvas. Default size of shadows in pixels. Set it to 0 to remove shadows. If true the bars will be presented horizontally (note: this can only be set for Bar Charts). If set to true shows the chart in stacked form (note: this can only be used to create Stacked Bar/Line Charts). Allows labels to be displayed beside bars or data points.

The following attributes should be defined within valueLabels

When set to true labels will be displayed. Defines the vertical alignment of the text. The available options are: 'top', 'above', 'below', 'middle' or 'bottom'. Defines the axis you wish to display the values of. The available options are: 'x' or 'y'. Defines the number of pixels the label should move to the left or right of its original position. Defines the number of pixels the label should move up or down from its original position. Defines the font size and font family the labels should use. Sets the text color of the labels. Should accept one argument that will contain the axis position of the associated data. The function can be used to change the text in the label and should do this by returning the desired text.

The following attributes should be defined within xaxis

If set to true displays the axis. Describes ticks of the axis. Set the start value of the x-axis. Set the end value of the x-axis. Defines the color of the axis lines. Sets the x-axis labels to a specified value. Defines how ticks are formatted. Defines how data is represented on the x-axis. Set as "top" or "bottom", specifies the position of the axis.

The following attributes should be defined within yaxis

If set to true displays the axis. Set the start value of the y-axis. Set the end value of the y-axis. Defines how ticks are formatted. Aligns the ticks of this axis with the ticks of the other axis. Set as "right" or "left", specifies the position of the axis.

The following attributes should be defined within grid

Specifies the color of the grid lines. If set to true enables interactivity in the grid. If set to true enables interactivity in the grid. Defines the width of the grid border. Specifies the colors of the ticks in an axis. Defines the border grid lines. If set to true this will prevent any x-axis labels from overlapping. Specifies the default minimum margin around the border. Used to make sure that points aren't accidentally clipped by the canvas edge. Specifies the background color or background gradient colors of the chart.

The following attributes should be defined within legend

If set to true shows the legend. The id or class name of the container to hold the legend. The number of columns for the legend. "ne" or "nw" or "se" or "sw", defines the position of the legend.

The following attributes should be defined within tooltip

Set to true to show the tooltip. Set to the name of a custom css class to style the tooltip. Set to false if you are using a custom css class. Set to true to have tooltip in fixed position. When set to false, the tooltip moves along with the mouse pointer. Values to shift tooltip position relative to the mouse pointer for x and y, negative values are allowed. Specifies the tooltip text e.g. "x : %x, y : %y" - displays x : x-value, y : y-value. Function can be used if you require more control over how the content is generated. Note that if you wish to access the axis labels rather than the number value (co-ordinates) assigned to them when using a function, you must use the "%" identifier followed by the axis (x or y) rather than the "xval" or "yval" parameters that are passed to the function. Provides the format of the x-axis Date.
