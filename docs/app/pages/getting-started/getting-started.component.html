Installation

UX Aspects components can be used in Angular projects (Angular or AngularJS 1.x), and those shown in the CSS section can be used in non-Angular projects. Details on installing and using the components are shown below. The installation instructions are broken down into four alternative methods as detailed in the following tabsets:

  1. Install Node.js, which provides the npm package manager.
  2. Install the UX Aspects package and its dependencies: npm install @ux-aspects/ux-aspects --save
  3. Install rxjs-compat for Angular 6 applications: npm install rxjs-compat --save
  4. Install the Angular CDK: npm install @angular/cdk --save

The UX Aspects resources (scripts, CSS, fonts, and images) can be downloaded from the GitHub releases page.
  1. Clone the GitHub repository.
  2. Install the dependencies: npm install
  3. Build the project: grunt build. The resources (scripts, CSS, fonts, and images) can be found in the dist folder.

If your application does not use Node or Webpack, you can download UX Aspects and copy the fonts, img, and styles directories to your application. The UX Aspects stylesheet can be applied from styles/ux-aspects.css by adding a link tag in the page header. This is the only required step for non-Angular sites.

Upgrading AngularJS Components

If the component is marked AngularJS in the documentation, it will need to be upgraded using Angular's UpgradeAdapter.

The components marked AngularJS can be included by importing @ux-aspects/ux-aspects/ng1/ux-aspects-ng1.js. Over the coming months, these will be replaced with Angular components, so it is recommended to set up the project as a hybrid AngularJS/Angular site. See the Angular Upgrade Guide for details on modifying a project to support Angular components. Components marked Angular in the documentation should be included via the import keyword and downgraded to be used in a hybrid site.
