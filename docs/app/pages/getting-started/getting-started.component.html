Installation

UX Aspects components can be used in Angular projects (Angular 4 or AngularJS 1.x), and those shown in the CSS section can be used in non-Angular projects. Details on installing and using the components are shown below. The installation instructions are broken down into four alternative methods as detailed in the following tabsets:



  1. Install Node.js, which provides the npm package manager.
  2. Install the UX Aspects package and its dependencies: npm install @ux-aspects/ux-aspects --save
  1. Install Node.js, which provides the npm package manager.
  2. Install TypeScript: npm install typescript --save-dev
  3. Install Angular 4 and other dependencies:
    npm install @angular/common @angular/compiler @angular/compiler-cli @angular/core @angular/forms @angular/http @angular/platform-browser @angular/platform-browser-dynamic @angular/platform-server @angular/router chart.js ngx-bootstrap ng2-charts @types/chart.js --save
  4. Install Bower: npm install -g bower
  5. Install the UX Aspects bower package: bower install ux-aspects#{{ version }}

The UX Aspects resources (scripts, CSS, fonts, and images) can be downloaded from the GitHub releases page.
  1. Clone the GitHub repository.
  2. Install the dependencies: npm install
  3. Build the project: grunt build. The resources (scripts, CSS, fonts, and images) can be found in the dist folder.

Upgrading Angular JS Components

If the component is marked AngularJS in the documentation, it will need to be upgraded using Angular's UpgradeAdapter.

The components marked AngularJS can be included by importing ng1/ux-aspects-ng1.js. Over the coming months, these will be replaced with Angular 4 components, so it is recommended to set up the project as a hybrid AngularJS/Angular 4 site. See the Angular Upgrade Guide for details on modifying a project to support Angular 4 components. Components marked Angular 4 in the documentation should be included via the import keyword and downgraded to be used in a hybrid site.
