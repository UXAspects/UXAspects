UX Aspects components can be used in Angular projects (Angular 4 or AngularJS 1.x), and those shown in the CSS section can be used in non-Angular projects. Details on installing and using the components are shown below.

Install with Bower

  1. Install Node.js, which provides the npm package manager.
  2. Install TypeScript: npm install typescript --save-dev
  3. Install Angular 4 and other dependencies:
    npm install @angular/common @angular/compiler @angular/compiler-cli @angular/core @angular/forms @angular/http @angular/platform-browser @angular/platform-browser-dynamic @angular/platform-server @angular/router chart.js ng2-charts @types/chart.js --save
  4. Install Bower: npm install -g bower
  5. Install the UX Aspects bower package: bower install ux-aspects#{{ version }}

Direct Download

The UX Aspects resources (scripts, CSS, fonts, and images) can be downloaded from the GitHub releases page.

Building from Source

  1. Clone the GitHub repository.
  2. Install the dependencies: npm install
  3. Build the project: grunt build. The resources (scripts, CSS, fonts, and images) can be found in the dist folder.

Using UX Aspects

Add the UX Aspects stylesheet to the header of the site. This is the only required step for non-Angular sites.

In an Angular 4 project

Import the required component(s) into the module that will be using them.

If the component is marked Angular 1 in the documentation, it will need to be upgraded using Angular's UpgradeAdapter.

In an AngularJS project

The components marked Angular 1 can be included by importing ng1/ux-aspects-ng1.js. Over the coming months, these will be replaced with Angular 4 components, so it is recommended to set up the project as a hybrid AngularJS/Angular 4 site. See the Angular Upgrade Guide for details on modifying a project to support Angular 4 components. Components marked Angular 4 in the documentation should be included via the import keyword and downgraded to be used in a hybrid site.
